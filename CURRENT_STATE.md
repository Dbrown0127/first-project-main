# Current State

## Executive Summary

Numberdle is a local-first arithmetic puzzle page. Players fill five rows with equations that match target results, while also using each number from `1` through `10` exactly once and every operation at least once. Rows can be valid before the full board is solved.

## Completed Features

- Random puzzle selection from static difficulty libraries.
- Difficulty selector for Normal, Hard, and Extreme puzzles.
- Current difficulty and puzzle ID display.
- Local completed-puzzle tracking so solved puzzles are skipped by difficulty on the same device.
- Five target-result rows with editable left-number, operation, and right-number slots.
- Compact custom number pad and operation picker that remain visible at the bottom of the puzzle while scrolling.
- Full-height Delete and Reset controls integrated into the two number rows.
- Desktop keyboard support for numbers and operations, with `0` mapped to `10`.
- Live row validation based on evaluated math results.
- Yellow row state for valid equations.
- Full green solved state only when row validation and global constraints pass.
- Number pad states for used, missing, and duplicate numbers.
- Operation picker states for used and missing operations.
- Reset button that clears slots and tracker state.
- Refreshing the page loads a new random puzzle.
- Local browser usage with no install or build step.
- Static puzzle library files generated by difficulty under `data/`.
- One-time generation script for rebuilding the static puzzle library.

## Current Architecture

- **Frontend:** Single-page vanilla HTML, CSS, and JavaScript in `index.html`.
- **Input model:** Puzzle rows store controlled `{ a, op, b }` values. Button-based slots feed parsed expression objects into row and global validation without native text inputs.
- **Puzzle data:** `data/normalPuzzles.js`, `data/hardPuzzles.js`, and `data/extremePuzzles.js` contain generated puzzle sets by difficulty and are loaded directly by the UI.
- **Difficulty keys:** The app now uses `normal`, `hard`, and `extreme` as the real difficulty identifiers.
- **Local state:** Completed puzzle IDs are stored in `localStorage` by difficulty under `numberdle.completedPuzzlesByDifficulty`. If a difficulty pool is exhausted, only that difficulty's completed list is cleared.
- **Legacy puzzle data:** `PuzzlesArray.js` still exists, but the current UI no longer uses it.
- **Generation script:** `generatePuzzles.js` creates the static difficulty files with a maximum of 5,000 puzzles per difficulty.
- **Current generated counts:** Normal has `5000` puzzles, Hard has `5000` puzzles, and Extreme has `4502` puzzles.
- **Backend:** None.
- **Build tooling:** None.
- **Hosting:** The page can run by opening `index.html` directly and is also linked from README at its GitHub Pages URL.

- There is no in-page "new puzzle" button yet; users refresh the page to get a new puzzle.

## Development Phase

The project is in the working prototype phase. The core puzzle loop, row validation, global validation, and trackers are functional, but the project is not yet ready for a polished public launch.
